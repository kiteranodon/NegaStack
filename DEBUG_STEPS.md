# デバッグ手順

## 現在の状態
- 白い画面が表示される
- コンソールにエラーメッセージはない
- アプリアイコンに通知バッジが1件表示されている

## 実施した修正
1. 古い通知を全てクリア
2. デバッグ用のシンプルな青い画面を追加
3. Firebase初期化をバックグラウンドスレッドに移動
4. 詳細なログ出力を追加

## 確認事項

### 1. Xcodeコンソールの確認
実機でアプリを起動後、以下のログが表示されるか確認してください：

```
========================================
🚀 NegaStack アプリ起動開始
========================================
🧹 古い通知をクリアしました
✅ 通知デリゲート設定完了
========================================
🚀 アプリ起動完了 - UI表示開始
========================================
✅✅✅ ContentViewが表示されました ✅✅✅
✅ メインスレッドでUI更新確認
🔵🔵🔵 デバッグ画面が表示されました 🔵🔵🔵
```

### 2. 画面の確認
**青い画面が表示されますか？**
- はい → StartScreenに問題がある可能性
- いいえ → UI描画自体に問題がある

### 3. 次のステップ

#### A. 青い画面が表示される場合
ボタンをタップして実際のアプリを起動してください。
StartScreenでエラーが発生する場合、コンソールにエラーが表示されます。

#### B. 青い画面も表示されない場合
以下を確認：
1. Xcodeの「Product」→「Clean Build Folder」を実行
2. シミュレータで動作するか確認
3. 実機を再起動
4. Xcodeを再起動

## トラブルシューティング

### ケース1: コンソールに何も表示されない
→ アプリがクラッシュしている可能性
- Xcodeの「Device and Simulators」でクラッシュログを確認

### ケース2: 白い画面だが、ログは表示される
→ UIレンダリングの問題
- 実機の再起動を試す
- iOS のバージョンを確認（iOS 15以降推奨）

### ケース3: Firebase関連のエラーが出る
→ GoogleService-Info.plistの問題
- ファイルがターゲットに追加されているか確認
- Bundle IDが一致しているか確認
